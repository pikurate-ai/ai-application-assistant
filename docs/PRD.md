# 제품 요구사항 문서 (Product Requirements Document)

## 프로젝트명
AI 기반 지원서 자동 작성 도우미 (AI Application Assistant MVP)

## 작성 일시
- 생성: 2025-11-22
- 최종 업데이트: 2025-11-22

## 1. 프로젝트 배경 및 목표

### 배경
사용자가 여러 지원서를 작성할 때 기존에 작성했던 내용을 재활용하고, 새로운 양식에 맞게 최적화하는 과정이 반복적이고 시간이 많이 소요됨.

### 목표
LLM을 활용하여 기존 지원서 데이터를 기반으로 새로운 지원서를 자동으로 생성하고, Google Docs로 출력하는 웹 서비스 MVP 개발.

### 제약사항
- **비용**: 모든 인프라 무료 티어 사용
- **개발자**: 비개발자 1인 (GitHub Copilot CLI 활용)
- **범위**: MVP - 핵심 기능만 우선 구현

## 2. 핵심 기능 명세

### 2.1 입력 기능
**F1: 새로운 지원서 양식 입력**
- 사용자가 새 지원서의 질문 항목들을 입력
- 전체 분량 제한 입력 (예: 5페이지 이내, 3000자 이내)
- 입력 방식: 텍스트 직접 입력 또는 파일 업로드 (TXT, DOCX)

**F2: 기존 지원서 데이터 입력**
- 여러 개의 기존 지원서 텍스트 입력
- 입력 방식: 텍스트 직접 입력 또는 파일 업로드
- 최대 5개까지 입력 가능 (MVP)

### 2.2 분석 및 생성 기능 (LLM)
**F3: 질문 의도 파악**
- 새로운 지원서의 각 질문이 요구하는 내용 분석
- 질문 카테고리 분류 (경험, 역량, 동기 등)

**F4: 내용 추출 및 재구성**
- 기존 지원서에서 각 질문에 적합한 내용 추출
- 새로운 질문 의도에 맞게 내용 재구성
- 문장 다듬기 및 일관성 유지

**F5: 중복 방지 및 분량 조절**
- 소제목 분석을 통한 내용 중복 방지
- 전체 분량 제한 내에서 각 항목별 분량 자동 조절
- 중요도에 따른 우선순위 설정

### 2.3 출력 기능
**F6: Google Docs 문서 생성**
- 생성된 내용을 구조화된 Google Docs 문서로 생성
- 사용자의 Google 계정으로 문서 저장
- 문서 링크 제공

**F7: 결과 미리보기**
- Google Docs 생성 전 웹에서 결과 미리보기
- 수정 요청 기능 (재생성)

## 3. 사용자 플로우

```
1. 사용자 접속 → 랜딩 페이지
2. Google 로그인
3. 새 지원서 양식 입력
4. 기존 지원서 데이터 입력
5. 분량 제한 설정
6. 생성 버튼 클릭
7. [백엔드] LLM 분석 및 생성
8. 결과 미리보기
9. Google Docs 내보내기
10. 문서 링크 확인
```

## 4. 기술 요구사항

### 4.1 LLM 프롬프트 설계
**핵심 프롬프트 구조:**
```
시스템 역할: 지원서 작성 전문가

입력:
1. 새 지원서 질문 목록
2. 기존 지원서 내용들
3. 분량 제한

작업:
1. 각 질문의 의도 파악
2. 기존 내용에서 관련 정보 추출
3. 새 질문에 맞게 재구성
4. 중복 제거
5. 분량 조절

출력:
JSON 형식의 구조화된 답변
```

### 4.2 Google Docs API 연동
- OAuth 2.0 인증
- Documents API 사용
- 문서 생성 및 포맷팅
- 권한 설정 (사용자 소유)

### 4.3 성능 요구사항
- LLM 응답 시간: 30초 이내
- 전체 프로세스: 1분 이내
- 동시 사용자: 10명 (MVP)
